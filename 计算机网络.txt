(𐅙)计算机网络的性能指标{
	速率：数据率或数据传输率或比特率（每s传输多少比特）
		速率的进制{一般速率的单位用小写
			1kb/s = 10^3b/s
			1Mb/s = 10^3kb/s = 10^6b/s
			1Gb/s = 10^3Mb/s = 10^6kb/s = 10^9b/s
			1Tb/s = 10^3Gb/s = 10^6Mb/s = 10^9kb/s = 10^12b/s
		}
		储存容量的进制{一般容量的大小单位用大写
			1Byte = 8Bit
			1KB = 1024B
			1MB = 1024KB
			1GB = 1024MB
			1TB = 1024GB
		}
	带宽：原指某个信号具有的频带宽度，即最高频率与最低频率之差，单位是赫兹（HZ）
		计算机网络中，带宽用来表示网络的通信线路传送数据的能力，通常是单位时间内从网络中的某一点到另一点所能通过的“最高数据率”。单位是“比特每秒”，b/s,kb/s,Mb/s,Gb/s，（可以理解为网络设备支持的最高的速度,是网络入口发送的速率，而不是信道上传输的速率）
	吞吐量：单位时间内，通过某个网络（信道，接口）的数据量（接收和发送的数据量的总和），单位b/s,kb/s,Mb/s等
		吞吐量受网络的带宽或网络的额定速率的限制

	时延：数据（报文/分组/比特流）从网络的一端传送到另一端所需的时间。也叫延迟或迟延，单位是s
		发送时延（传输时延）：从发送分组的第一个bit算起，到该分组的最后一个bit结束所需的时间
			发送时延 = 数据长度 / 信道带宽（发送速率）
		传播时延：电磁波传输一定距离所需的时间，取决于电磁波的传播速率和链路长度
			传播时延 = 信道长度/ 电磁波在信道上的传播速率
		排队时延：等待输出/输入链路可用（例如排队等待路由器处理）
		处理时延：处理数据所用的时间（路由器等）
		高速链路提高的是发送速率，减小了发送时延，如果采用不同的介质可以改变电磁波的传输速率，进而可以改变传播时延
	时延带宽积：（以比特为单位的链路长度，即“某链路现在有多少比特”）
		时延带宽积 = 传播速率 × 带宽
		单位：bit		s		 b/s
	往返时延RTT：（从发送方发送数据开始，带发送方收到接受到接受方的确认，及发出数据的第一个bit开始到接收到确认信息的第一个bit结束）
		RTT越大，在收到确认前，可以发送的数据越多
		RTT包括（不包括传输（发送）时延）
			往返时延： 传播时延 × 2
			末端处理时间：接受方收到发送方的数据并对数据进行处理之后发出确认信息
	利用率：
		信道利用率：
			链路有多长时间是有数据的
			信道利用率 = 有数据通过的时间 / （有 + 无）数据通过的时间
		网络利用率：
			信道利用率加权平均值

分层结构{
	为什么要分层：{
		发送文件的步骤：
			发送前激活数据链路
			告诉网络如何识别目的主机
			发起通信的主机要查明目的主机是否开始并正常连接到了网络
			发起通信的计算机要弄清目标主机的文件管理程序是否做好准备（是否可以接收，是否有足够的空间等）
			确保差错和意外可以解决
			…………
		发送文件等请求需要处理的东西繁多复杂，不便于管理，分层可以条理清晰地解决这些问题（将大问题转化为小问题）
	}

	实体：每一层当中的活动元素,第n层中的活动元素成为n层实体
	对等实体：同一层的实体
	协议：用以协助进行中网络上的对等实体之间进行数据交换的规定，要求叫做协议，协议是水平方向的，只有对等实体之间才存在协，
		协议的三要素：
			语法：规定传输数据的格式
			语义：规定所要完成的功能
			同步：规定各种操作的顺序
	接口：上层使用下层服务的入口，计算机网络中的接口是逻辑性的
	服务：下层为上层提供的功能调用，计算机网络中上层使用下层服务，下层为上层提供服务
	分层的基本原则：
		各个层次之间相互独立，每层只实现一种相对独立的功能
		每层之间的界限自然清晰，便于理解，尽量减少不同层次之间的交流
		结构上可分割，每层用最适合的技术实现
		保持下层对上层的独立性，上层单向使用下层提供的服务
		整个分层结构应该能促进标准化工作
	SDU数据服务单元：为完成用户所要求的功能所需传输的数据
	PCI协议控制信息：控制协议操作的信息
	PDU协议数据单元：对等层次之间传送的数据单位
	网络结构是从功能上描述的
	计算机网络体系结构简称网络体系结构是分层结构
	每层通过相应的协议完成该层的工作
	计算机网络体系结构是计算机网络各层及其协议的集合
}

(𐅙)OSI参考模型{
	目的：为了支持异构网络结构的互联互通
		国际标准化组织（ISO）于1984年提出开放系统互联（OSI）参考模型
		理论成功，市场失败
			失败原因：
				专家没有实际经验，类似于乌托邦
				太过复杂
				制定周期太长
				层次划分不很合理，有些功能在不同层次间重复出现

	OSI各层作用
					应用层---------资源子网{实现资源共享设备和软件的集合}(端到端)
					表示层---------资源子网(端到端)
					会话层---------资源子网(端到端)
					传输层---------（上下两个子网的接口(端到端)
	路由器----------网络层---------通信子网{各种传输介质，通信设备，相应的网络协议组成}(点到点)
	交换机，网桥----数据链路层-----通信子网(点到点)
	集线器，中继器--物理层---------通信子网(点到点)
	
	应用层：所有能和用户交互产生网络流量的程序（由此可知，记事本就不能算是在应用层）
		典型的应用层服务：
			应用（主要协议）
			文件传输（FTP）
			电子邮件（SMTP）
			万维网（HTTP）
			…………
	表示层：用于处理在两个通信系统中交换信息的表示方式（语法和语义）
		功能1：数据格式变换（翻译）
		功能2：数据加密和解密
		功能3：数据的压缩和恢复
		并没有主要协议，非要说的话主要协议为：JPEG，ASCII
	会话层：向表示层实体/用户进程提供建立连接并在连接上有序地传输数据(整个操作叫做会话，也是建立同步SYN)
		功能1：建立，管理，终止会话
		功能2：使用校验点可使会话在通信失效时从校验点/同步点继续恢复通信，实现数据同步（适用于传输大文件）
		主要协议：ADSP，ASP
	传输层：负责主机中两个进程的通信，即端到端的通信，传输单位是报文段或用户数据报
		功能1：可靠传输，不可靠传输
		功能2：差错控制
		功能3：流量控制（数据包传输速度问题，例如接受端一次能接受的数据是有上限的）
		功能4：复用分用
			复用：多个应用可以同时使用下面传输层的服务
			分用：运输层把接受到的信息分别交付给上面应用层中相应的进程
		主要协议：TCP，UDP
	网络层：主要任务是把分组从源端传到目的端，为分组交换网上的不同主机提供通信服务（网络层传输单位是数据报）（数据报过长时被切割为分组）
		功能1：路由选择（选择最佳路径）
		功能2：流量控制
		功能3：差错控制
		功能4：拥塞控制（如果所有结点都来不及接受分组，而要丢弃大量分组的话，网络就处于拥塞状态，因此要采取一定措施，缓解这种拥塞）
		主要协议：IP，IPX,ICMP,IGMP,APR,RAPR,OSPF
	数据链路层：主要任务是把网络层传下来的数据报组装成帧（数据链路层的传输单位是帧）
		功能1：成帧（定义帧的开始和结束）
		功能2：差错控制（帧错+位错）
		功能3：流量控制
		功能4：访问（接入）控制（控制对信道的访问）
		主要协议：SDLC，HDLC，PPP，STP
	物理层：主要任务是在物理媒体上实现比特流的透明传输（物理层的传输单位是比特）
		透明传输：指不管所传输的数据是什么样的比特组合，都应当能够在链路上传送
		功能1：定义接口类型
		功能2：定义传输模式（单工，半双工，双工）
		功能3：定义传输速率
		功能4：比特同步
		功能5：比特编码
		主要协议：Rj45,802.3
}

物理层基本概念{
	概念：
		物理层解决如何在连接各种计算机的传输媒体上传输数据比特流，而不是指具体的传输媒体
	物理层的主要任务：
		确定与传输媒体接口有关的一些特性	->	定义标准
			机械特性：定义物理连接的特性，规定物理连接时所采取的规格，接口形状，引线数目，引脚数量和排列情况
			电器特性：规定传输二进制时，线路上信号的电压范围，阻抗匹配，传输速率和距离限制等
			功能特性：指明某条线上出现的某一电平表示何种意义，接口部件的信号线的用途
			规程特性：定义各条物理线路的工作规程和时序关系
}
数据通信基础知识{
	典型的数据通信模型{
		主机---->调制解调器---->广域网---->调制解调器---->主机
		  数字信号		模拟信号	模拟信号	  数字信号

		------>|信源|------>|发送器|------>|传输系统|------>|接收器|------>|信宿|------>
		输入信息	输入数据	  发送的信号	  接收的信号	   输出数据		输出信息
	}
	数据通信的相关术语{
		通信的目的是传送信息
		数据：传送信息的实体，通常是有意义的符号序列
		信号：数据的电气/电磁的表现，是数据在传输过程中的存在形式
			数字信号：代表消息的参数取值是离散的
			模拟信号：代表消息的参数取值是连续的
		信源：产生和发送数据的源头
		信宿：接收数据的中点
		信道：信号的传输媒介，一般用来表示向某一个方向传送信息的介质，因此一条通信线路往往包含一条发送信道和一条接受信道（信道是有方向的）
		信道的划分：
			按传输信号分：模拟信道（传输模拟信号）和数字信道（传输数字信号）
			按传输介质分：无线信道（不需要物理上的有线介质）和有线信道（双绞线，光纤等）
	}
	三种通信方式：（从通信双方信息的交互方式看，可以有三种基本方式）
		单工通信：只有一个方向的通信，而没有反方向的交互，仅需要一条信道
		半双工通信：通信的双方都可以发送或接收信息，但任何一方都能同时发送和接收，需要两条信道
		双工通信：通信双方可以同时发送和接受数据，需要两条信道
	两种数据传输方式：
		串行传输：比特一个一个地排好队，发送给接收方（价格低，速度慢，适合远距离）
		并行传输：发送方同时发送多个比特，接受方同时接受多个比特（速度快，费用高，适合近距离）
}
码元，速率，带宽{
	码元：
		码元是指一个固定时长的信号波形（数字脉冲），代表不同离散数值的基本波形，是数字通信中数字信号的计量单位，这个时长内的信号称为k进制码元，而该时长称为码元宽度。当码元的离散状态有M个时（M大于2），此时码元为M进制码元。
		1码元可以携带多个比特的信息量，例如，在使用二进制编码时，只有两种不同的码元，一种代表0状态，另一种代表1状态
	速率：
		速率也叫数据率，是指数据的传输速率（发送速率），表示单位时间内传输的数据量，可以用码元传输数据速率和信息传输速率表示
			码元传输速率：别名码元速率，波形速率，调制速率，符号速率等，它表示单位时间内数字通信系统所传输的码元个数（也可以称为脉冲个数或信号变化的次数），单位是波特（baud），1波特表示数字通信每秒传输一个码元。这里的码元可以是多进制的，也可以是二进制的，但码元速率与进制数无关，1 Baud = 1 码元/s
		信息传输速率：别名信息速率，比特率等，表示单位时间内数字通信系统传输的二进制码元个数（即比特数），单位是比特/秒（b/s）
		关系：若一个码元携带n bit的信息量，则M Baud的码元传输速率所对应的信息传输速率为M × n bit/s
	带宽：表示在单位时间内从网络中的某一点到另一点所能通过的“最高数据率”，常用来表示网络的通信线路所能传输数据的能力，单位是b/s

}
(𐅙)奈氏准则和香农定理{
	信道带宽：原指某个信号具有的频带宽度，即最高频率与最低频率之差	
	失真：信号传递的过程中发生了变化，失去了真实性（与发送时的信号不一致）,失真小的数据仍可被正确识别，失真过大就无法被识别了
		码间串扰---失真的一种现象：
			接受端收到的信号波形是去了码元之间清晰的界限的现象（码元的传输速率过快，计算机分辨不出来信号的频率等所携带的信息）
		影响失真的因素：
			码元传输速率（码元传输速率越快，失真越严重）
			信号传输距离（距离越远，衰减越旧，失真越大）
			噪声干扰（干扰越大，失真越大）
			传输媒体质量（质量越差，失真越大）
	奈氏准则（奈奎斯特定理）（限制码元传输速率的上限）
		在理想低通（无噪声，带宽受限）条件下，为了避免码间串扰，极限码元传输速率为2W Baud，W是信道带宽，单位是Hz
		理想低通信道下的极限数据传输率：2Wlog2 V(b/s)
		V:几种码元/码元的离散电平数目
	根据奈氏准则可以得出：
		在任何信道中，码元传输的速率都是有上限的，若传输速率超过此上限，就会出现严重的码间串扰问题，使接收端对码元的完全正确识别成为不可能
		信道的頻带带宽（即能通过的信号高频分量越多），就可以用更高的速率进行码元的有效传输
		奈氏准则给出了码元传输速率的限制，但没有对信息的传输速率给出限制
		由于码元的传输速率受奈氏准则的制约，所以要提高数据的传输速率，就必须设法使每个码元能携带更多个比特的信息量，这就需要采用多元制的调制方法

	噪声存在于所有通信的电子设备和通信信道中，由于噪声随机产生，它的瞬时值会很大，因此噪声会使接收端对码元的判决产生错误，但是噪声的影响是相对的，若信号较强，那么噪声影响相对较小，因此噪声比就很重要
		信噪比 = 信号的平均功率 / 噪声的平均功率，常记为S/N(此比值无单位)
		信噪比（dB） = 10 log10（S/N）(用分贝（dB）作为计量单位，与上面的比值在数值上是等价的,只是在表示方式上有所不同)

	香农定理：在带宽受限且有噪声的信道中，为了不产生误差，信息的数据传输速率有上限值
		信道的极限数据传输速率 = W log2(1 + S/N)(b/s)
	由香农定理可知：
		信道的带宽或信道中的信噪比越大，则信息的极限传输速率就越高
		对于一定的传输带宽和一定的信噪比，信息传输速率的上限就可以确定了
		只要信息的传输速率低于信道的极限传输速率，就一定能找到某种方法来实现无差别的传输
		香农定理得出的为极限信息传输速率，实际信道能达到的传输速率要比他低不少
		如果信道带宽W或信噪比S/N没有上限（不可能），那么信道的极限信息传输速率也就没有上限
	可以根据两个公式的出的数据取最小值作为实际的极限
